# 编辑工作区标签重构计划

## 实施进度

- [x] 阶段一：编辑任务领域模型、稳定任务键、去重与任务键迁移
- [x] 阶段二：任务状态容器和独立的 Ant Design 工作区标签组件
- [x] 阶段三：编辑入口、草稿状态和任务激活恢复桥接
- [x] 阶段四：普通页面移除全局标签并按任务条件调整布局
- [x] 阶段五：删除旧 `useOpenPages` 实现和遗留样式

## 1. 背景

当前应用通过 `useOpenPages` 将所有访问过的路由页面加入全局标签栏，包括工作台、列表、报表、设置、详情和编辑页面。顶部主导航已经承担模块切换，因此全局标签栏同时存在以下问题：

- 普通浏览页面与顶部导航表达重复，占用持续性的垂直空间。
- 标签代表“访问历史”而不是“未完成工作”，业务含义不清晰。
- 页面数量持续增长后识别成本高，关闭和双击关闭属于隐性操作。
- 列表、详情和设置页面没有需要长期保留的本地状态，却与单据编辑任务获得同等权重。
- 单据编辑器已经具备草稿保存与恢复能力，但全局标签状态没有与草稿生命周期建立明确关系。

## 2. 目标

采用“普通页面无标签，编辑任务有工作区标签”的混合导航模型：

1. 顶部主导航负责业务模块切换。
2. Breadcrumb 与页面标题负责普通页面定位。
3. 仅新建、编辑、重新核定等未完成任务进入编辑工作区标签。
4. 编辑标签表达任务状态，而不是浏览历史。
5. 刷新、重新登录或意外关闭后，可以基于草稿恢复未完成任务。
6. 不改变现有业务保存接口、权限判断和草稿数据格式。

## 3. 非目标

- 不实现任意页面的浏览器式多标签。
- 不把详情、列表、报表或设置页面加入编辑工作区。
- 不新增后端持久化表。
- 不跨用户共享草稿或编辑任务。
- 不在本阶段实现多人协同编辑、记录锁或冲突合并。
- 不改变 TanStack Router 的路由结构。

## 4. 术语

| 术语 | 定义 |
|---|---|
| 普通页面 | 工作台、列表、详情、报表、对账、系统设置等浏览型页面 |
| 编辑任务 | 新建、编辑、重新核定等具有表单状态和保存动作的业务任务 |
| 工作区标签 | 仅用于切换编辑任务的 Ant Design Tabs |
| 草稿 | 当前 `module-editor-draft-storage` 保存的编辑快照 |
| 任务键 | 唯一标识一个编辑任务的稳定键 |

## 5. 信息架构

### 5.1 页面层级

```text
顶部主导航
  ├─ 模块切换
  ├─ 全局搜索
  └─ 用户菜单

页面上下文
  ├─ Breadcrumb
  ├─ 页面标题
  └─ 页面级操作

编辑工作区（有编辑任务时才显示）
  ├─ 编辑任务标签
  └─ 当前编辑器
```

### 5.2 显示规则

- 没有编辑任务时，不渲染标签栏，也不保留空白高度。
- 普通路由切换不创建工作区标签。
- 用户点击“新建”“编辑”“重新核定”等操作时创建或激活任务标签。
- 同一任务重复打开时定位到已有标签，不创建重复标签。
- 从编辑任务返回列表后，未关闭的任务仍可从工作区标签恢复。
- 工作区最多直接展示 6–8 个标签，超出部分交由 Ant Design Tabs 的“更多”菜单管理。

## 6. 状态模型

### 6.1 EditorTask

```ts
interface EditorTask {
  key: string
  moduleKey: string
  mode: 'create' | 'edit' | 'reconfirm'
  recordId: string
  path: string
  title: string
  status: 'clean' | 'dirty' | 'saving' | 'saved' | 'error' | 'readonly'
  updatedAt: number
  closable: boolean
}
```

该接口仅用于说明计划中的领域模型，实施时应根据现有类型体系确定最终位置和字段。

### 6.2 任务键

任务键必须稳定且可去重：

```text
{userKey}:{moduleKey}:{mode}:{recordId}
```

- 编辑已有记录：`recordId` 使用业务记录 ID。
- 新建记录：优先使用当前预分配 ID。
- 没有预分配 ID 时，使用草稿存储已经生成的稳定临时 ID。
- 禁止使用数组下标、页面标题或当前时间直接作为任务键。

### 6.3 状态含义

| 状态 | 视觉 | 行为 |
|---|---|---|
| clean | 无状态点 | 当前内容与已保存数据一致 |
| dirty | 主色圆点 | 存在未保存修改 |
| saving | Spin | 正在提交，禁止重复保存和直接关闭 |
| saved | 短暂成功状态 | 保存成功后转换为 clean 或关闭任务 |
| error | 错误图标 | 保存失败，保留表单和草稿 |
| readonly | 锁图标 | 任务不可继续编辑，只允许查看或关闭 |

## 7. 交互规则

### 7.1 创建与激活

1. 新建单据时创建 `create` 任务。
2. 编辑已有单据时创建 `edit` 任务。
3. 交付核定后的重新核定使用 `reconfirm` 任务。
4. 创建前先按任务键查询现有任务；存在则激活并导航到原路径。
5. 标签标题优先使用单号；新建任务显示“新建 + 单据类型”。

### 7.2 切换

- 单击标签切换当前任务并导航到对应编辑路由。
- 切换前立即刷新当前草稿快照，避免防抖定时器尚未写入。
- 各标签表单状态由草稿快照和当前路由编辑器恢复，不在标签组件内复制整份表单数据。
- 标签切换不得触发保存接口。

### 7.3 关闭

| 条件 | 关闭行为 |
|---|---|
| clean | 直接关闭 |
| dirty 且草稿写入成功 | 显示“保存草稿并关闭 / 放弃修改 / 取消” |
| saving | 禁止关闭并提示正在保存 |
| error | 显示“保留草稿并关闭 / 继续处理” |
| readonly | 直接关闭 |

“放弃修改”必须同时删除对应草稿；“保存草稿并关闭”只关闭任务，不调用业务保存接口。

### 7.4 保存成功

- 新建保存成功后，删除新建草稿。
- 若保存后进入详情或列表，关闭对应编辑任务。
- 若业务要求保存后继续编辑，则将任务从 `create` 转换为 `edit`，重新生成稳定任务键。
- 任务键迁移必须原子更新，避免同时存在“新建标签”和“已保存标签”。

### 7.5 刷新与恢复

- 应用启动时仅扫描当前用户的编辑草稿索引，不扫描完整草稿正文。
- 恢复入口放在工作区“恢复任务”菜单中，避免登录后自动打开大量标签。
- 用户选择恢复后读取草稿正文并创建任务。
- 无权限、记录已删除或状态不可编辑时，将任务标记为 readonly，并提供删除草稿入口。

## 8. 组件设计

### 8.1 顶部与普通页面

- 移除 `AppLayout` 中对全局 `AppPageTabs` 的无条件渲染。
- 普通页面使用现有顶部导航、Breadcrumb 和工作区标题。
- 重新计算 `--app-top-offset`，没有编辑任务时内容紧跟顶部栏。

### 8.2 EditorWorkspaceTabs

建议新增领域明确的组件：

```text
src/layouts/editor-workspace/
  EditorWorkspaceTabs.tsx
  EditorWorkspaceTabLabel.tsx
  EditorWorkspaceCloseDialog.tsx
  useEditorTasks.ts
  editor-task-storage.ts
  editor-task-types.ts
```

使用 Ant Design：

- `Tabs type="editable-card"`
- `more={{ trigger: 'click' }}`
- 自定义 `closeIcon` 表达状态
- `Modal` 或项目统一确认组件处理未保存关闭
- `Dropdown` 提供恢复草稿、关闭其他、关闭全部已保存任务

### 8.3 职责边界

| 组件/Hook | 职责 |
|---|---|
| `useEditorTasks` | 创建、去重、激活、关闭和恢复任务 |
| `editor-task-storage` | 保存轻量任务索引，不保存表单正文 |
| `module-editor-draft-storage` | 继续负责草稿正文 |
| `ModuleEditorWorkspace` | 上报 dirty/saving/error 状态和保存结果 |
| `EditorWorkspaceTabs` | 纯展示与用户操作转发 |
| `AppLayout` | 布局组合，不承载任务业务规则 |

## 9. 现有代码迁移

### 9.1 `useOpenPages`

- 停止作为全局页面标签的数据源。
- 第一阶段保留实现但不渲染，降低回滚风险。
- 新工作区稳定后删除 `OpenPage`、`closePage`、`updatePageTitle` 等无使用代码。
- 不直接把 `useOpenPages` 改名为 `useEditorTasks`，两者领域语义与生命周期不同。

### 9.2 `AppPageTabs`

- 当前已迁移到 Ant Design Tabs，可作为交互验证参考。
- 实施时重命名或新建 `EditorWorkspaceTabs`，避免继续表达“所有页面标签”。
- 删除普通页面自动加入、工作台固定标签等逻辑。

### 9.3 编辑器与草稿

- 在 `useModuleEditorWorkspace` 暴露稳定的编辑状态摘要。
- 草稿写入成功后通知任务状态，但不让标签层直接读取表单对象。
- 页面卸载、切换任务、浏览器隐藏时复用现有草稿立即写入能力。
- 保持现有按用户隔离、大小限制和异常处理。

## 10. 分阶段实施

### 阶段一：领域模型与测试

1. 新增 `EditorTask` 类型和任务键生成函数。
2. 使用 TDD 覆盖新建、编辑、重新核定任务去重。
3. 覆盖新建保存后任务键迁移。
4. 覆盖不同用户之间任务隔离。

### 阶段二：任务状态容器

1. 实现 `useEditorTasks`。
2. 支持 create、activate、updateStatus、close、restore。
3. 保存轻量任务索引到本地存储。
4. 为损坏索引、过期任务和权限变化提供降级逻辑。

### 阶段三：编辑器接入

1. 新建、编辑、重新核定入口创建任务。
2. `ModuleEditorWorkspace` 上报状态变化。
3. 标签切换前执行草稿立即写入。
4. 保存成功、取消和放弃修改时同步任务生命周期。

### 阶段四：布局替换

1. 普通页面停止创建全局标签。
2. 仅存在编辑任务时渲染 `EditorWorkspaceTabs`。
3. 调整顶部偏移和内容高度。
4. 增加恢复任务、关闭其他和关闭全部已保存任务。

### 阶段五：清理

1. 删除未使用的 `useOpenPages` 和旧页签样式。
2. 删除基于访问历史的页面标题更新逻辑。
3. 更新中英文文案与类型声明。
4. 更新架构和用户操作文档。

## 11. 测试计划

### 11.1 单元测试

- 任务键生成和去重。
- create 保存后迁移为 edit。
- dirty、saving、error、readonly 状态转换。
- 关闭回退路径。
- 草稿索引损坏恢复。
- 用户隔离。
- 最大展示数量与更多菜单。

### 11.2 组件测试

- 无任务时标签栏不渲染。
- 新建或编辑后出现标签。
- 重复打开同一任务只激活原标签。
- dirty 标签关闭出现三选项确认。
- saving 标签不可关闭。
- overflow 使用 Ant Design 更多菜单。
- 键盘可以聚焦、切换和关闭标签。

### 11.3 集成测试

- 列表 → 编辑 → 修改 → 切换任务 → 返回后内容保留。
- 刷新浏览器 → 从恢复入口恢复任务。
- 新建保存成功 → 标签关闭或迁移。
- 交付核定 → 重新核定 → 保存 → 状态正确。
- 权限被撤回后恢复旧草稿。
- 当前记录被删除后恢复旧草稿。

### 11.4 视觉与响应式

- 1440px：直接展示 6–8 个任务。
- 1024px：标签缩短并进入更多菜单。
- 720px：仅显示当前任务和更多菜单。
- 深色主题下状态图标、选中边框和焦点环清晰。
- 200% 缩放下不遮挡保存与关闭操作。

## 12. 验收标准

- 普通页面切换不会新增标签。
- 没有编辑任务时，标签栏完全消失且内容区高度正确。
- 同一记录、同一模式、同一用户最多存在一个任务。
- 未保存修改不会因切换标签、关闭标签或刷新页面丢失。
- dirty 任务不能无确认丢弃。
- saving 任务不能重复保存或关闭。
- 新建保存后不会残留重复标签和孤立草稿。
- 标签溢出由 Ant Design Tabs 管理。
- 键盘操作和焦点状态符合现有可访问性要求。
- 不新增后端表，不改变现有业务保存接口。
- 现有编辑器、草稿、交付核定和权限测试全部通过。

## 13. 风险与缓解

| 风险 | 缓解措施 |
|---|---|
| 标签状态与草稿状态不同步 | 标签只保存摘要；正文始终以草稿存储为准 |
| 新建保存后任务键变化 | 使用显式任务迁移操作并编写原子迁移测试 |
| 关闭时防抖草稿尚未落盘 | 切换和关闭前强制 flush |
| 恢复任务时权限或记录状态变化 | 恢复前重新校验，降级为 readonly |
| 标签移除后用户找不到历史页面 | 使用 Breadcrumb、顶部导航和全局搜索承担定位 |
| 大量草稿导致启动变慢 | 启动仅读取轻量索引，按需读取正文 |
| 与当前导航重构重叠 | 先稳定 Menu/Tabs 视觉，再替换标签数据源和显示条件 |

## 14. 回滚策略

- 阶段四之前保留 `useOpenPages`，可通过布局配置重新显示旧标签栏。
- 新任务索引与旧草稿正文使用不同存储键，回滚不会破坏草稿。
- 不修改后端和数据库，因此回滚仅涉及前端布局与本地索引。
- 清理旧实现必须放在最终独立提交，确保可以单独回退。
